 🚀 Visão Geral
Este framework representa o estado da arte em automação de testes End-to-End (E2E). Desenvolvido por **Luciana Orli Fonseca da Cruz**, o projeto foca em alta performance, escalabilidade e observabilidade, utilizando o ecossistema **Python + Playwright + BDD**. 

A arquitetura foi projetada para ser **AI-Ready**, permitindo a transição imediata para validações de modelos de linguagem (LLMs) e automação inteligente, unindo a robustez do QA tradicional à vanguarda da Engenharia de IA.

🛠️ Stack Tecnológica & Diferenciais
 **Engine:** Playwright para automação de browser ultra-rápida (Headless/Headed).
 **Metodologia:** BDD (Behavior Driven Development) com Pytest-BDD para alinhamento total com requisitos de negócio.
 **Reporting:** Allure Framework para dashboards executivos e análise detalhada de causa raiz.
 **CI/CD:** Estrutura preparada para GitHub Actions com feedback contínuo.

 🏗️ Engenharia de Software (SDET Focus)
 **Page Object Model (POM):** Implementação rigorosa para garantir manutenibilidade e desacoplamento total.
 **Gherkin Syntax:** Requisitos técnicos traduzidos para linguagem de negócio, facilitando o Shift-Left testing.
**Clean Code:** Padrões de codificação voltados para evitar débito técnico em suítes de longa duração.

 🤖 Especialização em IA (Roadmap de Engenharia)
Este framework não é apenas funcional, ele é visionário:
* **LLM Validation:** Estrutura pronta para integração com bibliotecas de avaliação de IA (como DeepEval e RAGAS).
* **Robustness Testing:** Planejamento para testes de segurança e robustez em modelos generativos.
* **Data-Driven AI:** Automação baseada em dados para cobrir cenários complexos de Machine Learning e fluxos de IA.
